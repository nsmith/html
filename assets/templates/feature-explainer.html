Smart Rate Limiting

Protect your API from bursts and abuse with per-key request budgets that refill over time — no extra infrastructure, no guesswork.

Overview

At a glance, here is what this feature gives you.

Token-bucket budgets

Each client gets a refillable budget so steady traffic always passes.

Abuse protection

Bursts are smoothed out and bad actors are throttled automatically.

Clear feedback

Standard headers tell callers exactly how much budget remains.

How it works

Expand each step to see the details. The flow runs on every request.

1 Identify the caller

Every incoming request is mapped to a bucket key — typically the API token, but it can be an account ID or client IP. Requests without a key fall into a shared anonymous bucket. 

key = request.api_token ?? request.ip
2 Check the budget

Each bucket holds a number of tokens. A request costs one token. If at least one token is available, the request proceeds and the count is decremented.

3 Refill over time

Tokens regenerate at a fixed refill rate up to the bucket's maximum. This lets short bursts through while keeping the long-run average within your limit.

4 Respond

If the budget is exhausted, the API returns 429 Too Many Requests with a Retry-After header. Otherwise the response carries the remaining count.

Configuration

Set limits per route. Defaults apply when a field is omitted. 

rate_limit:
  default:
    capacity: 100      # max tokens in a bucket
    refill_per_sec: 5  # tokens added each second
  routes:
    "POST /upload":
      capacity: 10
      refill_per_sec: 1

Frequently asked questions

What happens when a client hits the limit?

They receive a 429 response with a Retry-After header indicating how many seconds until a token is available again.

Does this work across multiple servers?

Yes. Buckets are stored in a shared backend, so limits are enforced consistently no matter which instance handles the request.

Can I set different limits for trusted clients?

Absolutely — assign a custom bucket capacity per API key or tier to give partners higher budgets.
